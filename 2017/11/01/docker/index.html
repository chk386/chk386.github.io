도커란 무엇인가?(공홈)

왜 씀?

08~09년 쯤 솔루션 업체에서 근무할때 일이다.

오라클 DB를 사용하는 솔루션이였고 고객사에 문제가 발생시 고객사 오라클 DB를 dump를 떠달라고 사이트 직원에게 부탁하여 로컬로 복사한다. 그리곤 고객사 오라클 버전(8i, 9g, 10g)과 동일한 버전의 오라클을 로컬에다 설치 후 import작업을 수십번은 한듯 하다. 윈도우PC에 오라클 설치 거지 같은거 다들 알지? 더군다나 여러버전의 오라클을 윈도우PC에 설치하면 피씨가 뻗어버려서 항상 새로 깔고 지우고 import하고 아주 쌩노가다였지..

우리 도커는 저런 썡노가다 작업을 획기적으로 줄여준다(=야근을 줄 일수 있다). 도커 클라스 ㅇㅈ?
도커는 virtual machine처럼 OS레벨의 가상화가 아니라 Linux Container(최근엔 LibContainer)를 이용하기 때문에 이미지 용량과 자원관리 측면에서 대단히 효율적이라네..

딱 봐도 알것지? Guest OS가 없어서 가상화 기술보다 가볍고 빠르다.
docker는 docker engine 위에 application에서 사용할 binary & library만 올라간다.
덕분에 이미지 용량도 작다. 윈도우나 리눅스 이미지 용량을 보면 알수 있지?

어쨋든 가상화라 불안하지 않나?

레퍼런스

구글의 모든 서비스들은 컨테이너로 동작한다. 약 20억 개 컨테이너를 구동중

만약 님이 사내를 도커 도입을 하려는데 태클을 거는 개발자(팀장포함)가 있다면 그냥 그 회사 그만두고 이직 ㄱㄱ!
님이 재직중인 회사가 운영하는 서비스보다 1000000000만배 큰 구글님이 쓰고 있다. 무슨 말이 필요한가?

단점

  • container가 하나 뚫리면 host까지 뚫릴수 있다. -> 관리를 잘해야한다. (가상화의 경우 하나의 vm이 해킹을 당해도 다른 vm에 영향을 줄수 없다)
  • host OS와 다른 OS의 container를 올릴수 없다. windows서버에서 linux서버 컨테이너를 불가 -> 어차피 윈도우 서버는 안쓰지?

docker를 어따 쓰지?

  • 팀에서 쓸 개발환경 이미지 공유
  • 서버 어플리케이션 설치 이미지(빌드된 소스m 환경설정 파일 등)
  • auto scaling
  • MSA(micro service architecture)

Command Line

docker를 설치하면 client와 server가 따로 설치되며 쉘에서 docker명령을 실행하면 docker client가 server로 명령어를 전달하고 리턴받는 형태

docker [Options] 설명

Option 설명
-d detached mode 흔히 말하는 백그라운드 모드
-p 호스트와 컨테이너의 포트를 연결 (포워딩)
-v 호스트와 컨테이너의 디렉토리를 연결 (마운트)
-e 컨테이너 내에서 사용할 환경변수 설정
-name 컨테이너 이름 설정
-rm 프로세스 종료시 컨테이너 자동 제거
-it -i와 -t를 동시에 사용한 것으로 터미널 입력을 위한 옵션
-link 컨테이너 연결 [컨테이너명:별칭]

MYSQL를 띄워보자

docker 컨테이너 로그 확인

로그는 stdout, stderr만 가능
파일로 로그를 쌓는 컨테이너의 경우 표준출력으로 변환할것

docker 컨테이너에서 명령 실행

docker volume mount

mysql이나 redis를 file로 관리하는 경우 해당 컨테이너를 업데이트 하려면 삭제 후 pull(run)을 해야한다. 헐 저장된 DB 데이터도 날라감????
host(서버 또는 내 로컬 맥북) 물리 디렉토리에 컨테이너에서 사용하고 있는 디렉토리를 마운트 해주면 이미지가 날라가도 데이터는 저장할수 있다.

docker compose

docker 컨테이너 실행시 많은 옵션, 환경변수넣고, volume마운트 해야하고 힘들지 않습니꽈???
docker compose를 이용하면 yaml방식으로 명령어를 관리하며 한번 셋팅해놓으면 쉽게 docker 컨테이너를 띄울수 있음.
사실 쉘을 만들어서 docker cmd를 관리하는 방법도 있으나 compose가 훨씬 편하다. compose써라.. 2번 써라
